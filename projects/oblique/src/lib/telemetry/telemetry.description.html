The Oblique front-end framework includes a telemetry feature that collects usage information. It's important that the Oblique team understands which modules are used so it can be decided on where to invest it's limited resources.

The collected data are linked to the project name, version and homepage configured in the package.json file. Only the Oblique team and it's superiors have access to the collected data.

When is data collected?

Data is collected in development mode only. That is if you build in angular production mode, telemetry is turned off.

Ensure that you deploy only production builds of your Oblique project outside of your local development environment.

ng build --prod=true

What is collected?

The telemetry records the usage on module basis. While loading one of the oblique modules, its usage is recorded. Additionally to this, your application name, version and homepage are also recorded.

  • Oblique module names, the Oblique version and the theme used.
  • Application name, version and homepage (from your package.json).

Selective module usage

Don't load ObliqueModule, as it loads all oblique modules. Instead carefully choose the modules that you need and only import those. As an added bonus you will have a smaller bundle because angular build will tree shake the unused modules for you ;-).

Do selectively load the needed modules
@NgModule({
	imports: [
		...
		ObSpinnerModule,
		ObMasterLayoutModule,
		ObNotificationModule,
		...
	]
})
Don't load the ObliqueModule
@NgModule({
	imports: [
		...
		ObliqueModule
		...
	]
})
